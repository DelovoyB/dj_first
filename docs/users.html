test_create_user() → None[source]

Test that a user can be created with valid password, last login, is_superuser, username, first name, last name, email, is_staff, is_active, date_joined, image and phone number.

The full_clean method is called to ensure that the user is valid according to the model’s field definitions. The save method is then called to persist the user to the database.

The test then checks that the user has been created with the expected password, last login, is_superuser, username, first name, last name, email, is_staff, is_active, date_joined, image and phone number.
